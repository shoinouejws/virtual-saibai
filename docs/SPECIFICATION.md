# バーチャル栽培（Virtual Saibai） — 仕様・要件定義書

## 1. プロダクト概要

### 1.1 コンセプト

AIを活用し、様々な作物（野菜・果実）をバーチャルに栽培できるシミュレーションアプリ。
ユーザーは畑を耕し、種を植え、水や肥料を与えながら作物を育て、収穫を楽しむことができる。

**最大の特徴:** 収穫した作物は実際にユーザーのもとへ届けられる。バーチャルな栽培体験がリアルな報酬につながるユニークなサービスモデル。

### 1.2 プロダクトのゴール

- **フロントエンドプロトタイプ**の構築（栽培画面のみ）
- GitHub Pages で公開し、URLから誰でもアクセスできる形式にする

### 1.3 時間システム

- 1日 = 1ターン。現実の日付変更に連動して日次処理が実行される
- プレイヤーは毎日好きなタイミングでアクションを実行し、日付が変わるとパラメーター評価・ステージ遷移が行われる
- 未ログイン日がある場合は、未ログイン日数分の日次処理をすべて反映する
- 栽培上の時間変化（季節など）は現実時間とは別にゲーム内で管理する
- デバッグ用に時間経過を手動操作できる機能を用意する（将来的にはアプリ内課金アイテム化も検討）

### 1.4 対象ユーザー

- カジュアルゲームを楽しむ一般ユーザー
- 農業・栽培に興味のあるユーザー

## 3. 機能要件

### 3.1 栽培画面（メイン画面）

アプリのメイン画面。畑の状態と作物の成長を視覚的に表示し、ユーザーが栽培アクションを実行できる。

#### 3.1.1 画面構成

```
┌─────────────────────────────────────┐
│          ヘッダーエリア              │
│   アプリ名 / ステータス情報           │
├─────────────────────────────────────┤
│          畑・作物表示エリア           │
│   （畑のグリッド + 作物のビジュアル）  │
│   ※ マスをタップ → セル詳細ページへ   │
├─────────────────────────────────────┤
│  [🛒 ショップへ] [⚡ イベント]       │
└─────────────────────────────────────┘
```

メイン画面は畑全体の俯瞰ビュー。各マス（FarmCell）をタップすると、そのマス専用のセル詳細ページ（`/cell/:id`）へ遷移する。

#### 3.1.2 セル詳細ページ

FarmPage のマスをタップすると遷移する、特定セルの栽培状況と操作を集約したページ。
ルートは `/cell/:id`（`:id` はセルの `FarmCellState.id`）。

```

├─────────────────────────────────────┤
│   ステージ進行ロードマップバー        │
│   🌱→🏠→🍃→💮→🌸→🫐→🍓→✂️       │
├─────────────────────────────────────┤
│   デイリーアドバイス表示エリア        │
├─────────────────────────────────────┤
│       作物表示エリア（大きめ）        │
│   （選択マスの作物画像を拡大表示）    │
├─────────────────────────────────────┤
│   パラメーター表示エリア              │
│   （株の元気/水分/栄養/雑草/害虫/病気）│
├─────────────────────────────────────┤
│        アクションボタンエリア         │
│ （ステージに応じて動的に切替）        │
└─────────────────────────────────────┘
```

**表示内容:**

### 3.2 ユーザーアクション

#### 3.2.1 アクション体系

アクションは作物の成長モデルによって体系が異なる。

**シンプルモデル（トマト）:** 耕す → 植える → 水やり/肥料 → 収穫の固定フロー。
**アドバンスドモデル（いちご）:** 8つの栽培ステージごとに利用可能なアクションが動的に切り替わる。詳細は `strawberry_progress_design.md` のステージ別行動表を参照。

#### 3.2.2 アクション共通仕様

| 項目 | 内容 |
|------|------|
| 回数制限 | 1日のアクション回数は **無制限**。パラメーターの上限（0〜100）が自然な行動制限として機能する |
| 即時反映 | 耕す・畝を作る等の物理的作業は即時に状態へ反映される |
| 日次反映 | 植物の成長・パラメーター評価は日次処理で反映される |
| 強度選択 | 水やり・追肥など量に関わるアクションは強度（少なめ/ふつう/たっぷり）を選択できる |

#### 3.2.3 アクション一覧（共通）

| アクション | 対象 | 消費アイテム | 備考 |
|-----------|------|------------|------|
| 耕す | 未使用マス | なし | マスが「耕された土」に変化 |
| 植える | 耕されたマス | なし | 作物選択UI表示。いちごは8ステージモデル開始 |
| 水やり | 栽培中のマス | なし | 強度選択あり（少なめ/ふつう/たっぷり） |
| 肥料 | 栽培中のマス | 肥料 | 強度選択あり |
| 除草 | 栽培中のマス | なし | 手作業 |
| 葉の整理 | 栽培中のマス | なし | 手作業 |
| 害虫対策 | 栽培中のマス | 殺虫剤 | ステージに応じて表示 |
| 病気対策 | 栽培中のマス | 殺菌剤 | ステージに応じて表示 |
| 温度調整 | 栽培中のマス | 防寒・遮光シート | ステージに応じて表示 |
| 受粉補助 | 開花期のマス | なし | 手作業。いちごの開花期のみ |
| 摘花・摘果 | 開花期〜のマス | なし | 手作業。数を減らし品質を上げる |
| 収穫 | 収穫可能マス | なし | 収穫結果確定。マスは空き地に戻る |

> 各アクションのステージごとの効果値は `strawberry_progress_design.md` を参照。

**選択可能な作物（プロトタイプ）:**

| 作物 | 成長モデル | 成長段階数 | 備考 |
|------|-----------|-----------|------|
| トマト | シンプル | 4 | 旧モデル維持（要再検討） |
| いちご | アドバンスド | 8 | 詳細は `strawberry_progress_design.md` 参照 |

> **プロトタイプでの扱い:** 実際の配送処理は実装しない。収穫時に演出を表示し、収穫履歴として記録するのみ。

### 3.3 収穫リスト機能

ヘッダーの収穫アイコンから遷移する独立ページ（`HarvestListPage`）。

#### 3.3.1 画面構成

```
┌──────────────────────────────────────┐
│  [← 畑に戻る]        収穫リスト       │
├──────────────────────────────────────┤
│  収穫した作物の一覧                    │
│                                      │
│  🍅 トマト × 4玉  2026/03/16  [交換]  │
│  🍓 いちご × 15粒 2026/03/15  [交換]  │
│  🍅 トマト × 6玉  2026/03/14  [交換]  │
│                                      │
│  （収穫なしの場合はその旨表示）         │
└──────────────────────────────────────┘
```

#### 3.3.2 交換個数の表示

- 各収穫レコードに「交換できる個数」を表示する（例: 「🍅 トマト × 4玉」「🍓 いちご × 8粒」）
- 交換個数の算出方法は作物の成長モデルによって異なる

**シンプルモデル（トマト）:**
- 交換個数は収穫時にランダムで決定（3〜6玉）

**アドバンスドモデル（いちご）:**
- 交換個数は栽培結果に基づいて算出される
- `交換個数 = 実の数 × 品質ランク補正`（端数切り捨て、最低1）

| 品質ランク | 補正倍率 |
|-----------|---------|
| A | ×1.2 |
| B | ×1.0 |
| C | ×0.8 |
| D | ×0.5 |

> 収穫結果の詳細（品質スコア計算・評価コメント・改善ヒント）は `strawberry_progress_design.md` セクション5を参照。

#### 3.3.3 交換ボタン

- 各収穫済み作物の行に「交換」ボタンを配置する
- プロトタイプのため、ボタン押下時は「交換申請を受け付けました」等のメッセージを表示するのみ（実際の配送処理なし）
- 将来的には EC / 物流システムと連携し、実際の作物を届ける

## 4. データモデル

プロトタイプ段階では `localStorage` にJSON形式で保存する。

### 4.1 ゲーム状態（GameState）

```typescript
interface GameState {
  fertilizer: number;           // 肥料の在庫数
  insecticide: number;          // 殺虫剤の在庫数
  fungicide: number;            // 殺菌剤の在庫数
  temperatureSheet: number;     // 防寒・遮光シートの在庫数
  farmSize: number;             // 畑のマス数
  cells: FarmCellState[];       // 各マスの状態
  harvestLog: HarvestRecord[];  // 収穫履歴
  lastLoginDate: string;        // 最終ログイン日（ISO 8601 日付）
  currentGameDate: string;      // ゲーム内現在日（ISO 8601 日付）
}

interface HarvestRecord {
  crop: CropType;
  harvestedAt: string;          // ISO 8601 日時
  exchangeQuantity: number;     // 交換できる作物の個数
  // アドバンスドモデル用（いちご）
  qualityRank?: 'A' | 'B' | 'C' | 'D';
  qualityScore?: number;
  fruitCount?: number;          // 収穫個数
  totalWeight?: number;         // 総重量(g)
  sweetness?: number;           // 甘さ
}
```

### 4.2 畑マスの状態（FarmCellState）

作物の成長モデルに応じて `cropState` の型が切り替わるユニオン型を採用する。

```typescript
interface FarmCellState {
  id: number;
  status: CellStatus;
  crop: CropType | null;
  cropState: SimpleCropState | AdvancedCropState | null;
}

// シンプルモデル（トマト用）
interface SimpleCropState {
  modelType: 'simple';
  growthPoints: number;
  maxGrowthPoints: number;
  growthStage: number;
  maxGrowthStage: number;
}

// アドバンスドモデル（いちご用）
// パラメーター詳細は strawberry_progress_design.md セクション2参照
interface AdvancedCropState {
  modelType: 'advanced';
  cultivationStage: number;     // 1〜8（現在のステージ番号）
  daysInStage: number;          // 現在ステージに入ってからの日数
  health: number;               // 健全度 0〜100
  moisture: number;             // 水分 0〜100
  nutrition: number;            // 栄養 0〜100
  stress: number;               // ストレス 0〜100（乾燥・過湿・高低温・強風・多肥で増加。水分・栄養が適正なら自然回復、詳細は strawberry_progress_design.md §3「ストレスの増減」参照）
  pestRisk: number;             // 害虫リスク 0〜100
  diseaseRisk: number;          // 病気リスク 0〜100
  weedAmount: number;           // 雑草量 0〜100
  rotRisk: number;              // 腐りやすさ 0〜100
  stageProgress: number;        // ステージ進行度 0〜100
  flowerCount: number;          // 花の数
  fruitCount: number;           // 実の数
  fruitSize: number;            // 実の大きさ 0〜100
  sweetness: number;            // 甘さ 0〜100
  coloring: number;             // 色づき 0〜100
  qualityDamage: number;        // 品質被害蓄積 0〜100
  qualityBonus: number;         // 品質補正
  overripeRisk: number;         // 熟しすぎリスク 0〜100（ステージ8で使用）
  // ステージ固有フラグ
  isTilled: boolean;            // 耕し済み
  hasRidge: boolean;            // 畝あり
  hasMulch: boolean;            // ビニールシートあり
  isPlanted: boolean;           // 定植済み
  rootEstablishment: number;    // 根付き度 0〜100（ステージ2で使用）
}
```

### 4.3 列挙型

```typescript
type CellStatus = 'empty' | 'tilled' | 'planted' | 'growing' | 'harvestable';

type CropType = 'tomato' | 'strawberry';

type CropModelType = 'simple' | 'advanced';

// いちごの栽培ステージ（アドバンスドモデル）
type StrawberryCultivationStage = 1 | 2 | 3 | 4 | 5 | 6 | 7 | 8;
// 1: 栽培準備期, 2: 定植・活着期, 3: 葉成長期, 4: 花芽形成期,
// 5: 開花期, 6: 果実肥大期, 7: 成熟期, 8: 収穫可能期
```

### 4.4 作物定義（CropDefinition）

```typescript
interface CropDefinition {
  type: CropType;
  name: string;
  modelType: CropModelType;
  maxGrowthPoints?: number;   // シンプルモデル用
  growthStages: number;       // 表示用段階数（トマト:4, いちご:8）
```

### 4.5 いちご（アドバンスド）— 雑草・マルチング・除草・健全度

#### 雑草量（`weedAmount`）

- 範囲は 0〜100。日次処理で増える（自然減少はない）。耕す・除草プレイヤーアクションでのみ減る。
- **日次増加量**は栽培ステージごとの `weedDelta` を `applyNaturalChanges` で加算する（詳細は `strawberry_progress_design.md` の日次自然変化に準拠）。

#### マルチング（`hasMulch`）と雑草

- マルチ敷設後は、その日の **雑草増加分 `weedDelta` に対して** `floor(weedDelta × 0.5)` を適用する（増加量を半分に切り捨て）。
- 設計上の意図は「雑草の発生・伸長を約 50% 抑える」こと。元の `weedDelta` が 1 のときは 0 になり得るため、**常にちょうど 50% にはならない**（小さい増加分はゼロに丸められる）。

#### 除草アクションの扱い

- **全栽培ステージ（1〜8）**で「除草」を選べること。いちご栽培では開花以降も畦畔・株間の手除草が行われるため、ステージ後半でも対応可能とする。
- 効果: `weedAmount` を 15 減らす（0 未満にはならない）。耕す（準備期）による雑草減は別アクション（`-10`）。

#### 健全度（`health`）と雑草

- 日次の `updateHealth` において、雑草は **養分・水分の競合**として健全度に影響する。
- **閾値**: `weedAmount >= 60` のとき健全度 `-2` / 日、`40 <= weedAmount < 60` のとき `-1` / 日（他の要因による加減算と加算される）。
- 雑草は `stress` を直接は変えないが、健全度低下を通じて間接的にステージ進行（健全度条件があるステージ）に影響し得る。

#### その他（ステージ進行度）

- ステージ3（葉成長期）では `weedAmount > 50` のとき、その日のステージ進行度加算が `-8` される（既存仕様）。

## 5. 画面遷移・状態遷移

### 5.1 画面遷移

```
栽培画面（メイン） ⇄ セル詳細ページ（/cell/:id）
栽培画面（メイン） ⇄ ショップページ
栽培画面（メイン） ⇄ 収穫リストページ
栽培画面（メイン） ⇄ イベントメニューページ
```

5画面構成。React Router（HashRouter）を使ったページ遷移。
FarmPage でマスをタップするとセル詳細ページへ遷移し、個別マスの栽培操作を行う。

### 5.2 畑マスの状態遷移

#### シンプルモデル（トマト）

```
[empty] ──(耕す)──▶ [tilled] ──(種を植える)──▶ [planted]
                       ▲                          │
                       │                    (水やり/肥料)
                       │                          ▼
                  (収穫する)◀── [harvestable] ◀── [growing]
                                              (成長ポイント
                                               が最大に到達)
```

#### アドバンスドモデル（いちご）

```
[empty] ──(耕す)──▶ [tilled]
                       │
                  (栽培開始)
                       ▼
              [growing] (8ステージ)
              ┌───────────────────────────────────────────┐
              │ S1:準備 → S2:定植 → S3:葉 → S4:花芽 →     │
              │ S5:開花 → S6:肥大 → S7:成熟 → S8:収穫可能  │
              └───────────────────────────────────────────┘
                       │
                  (収穫する)
                       ▼
                    [empty]
```

> 各ステージの遷移条件・アクション・イベントの詳細は `strawberry_progress_design.md` セクション4を参照。
> 収穫後はマスが空き地（empty）に戻る。

### 5.3 成長ロジック

#### シンプルモデル（トマト）

| アクション | 加算量 |
|-----------|--------|
| 水やり | +1 pt |
| 肥料 | +3 pt |

| 作物 | 最大成長ポイント | 成長段階数 | 各段階の区切りポイント |
|------|---------------|-----------|---------------------|
| トマト | 12 pt | 4 | 3 / 6 / 9 / 12 |

1. 成長ポイントが各段階の区切りポイントに達するたびに `growthStage` が1つ進む
2. `growthStage` が `maxGrowthStage`（4）に達すると `harvestable` 状態になる
3. 成長ポイントは上限（`maxGrowthPoints`）を超えない

#### アドバンスドモデル（いちご）

成長ポイント方式ではなく、17パラメーターの日次評価による栽培シミュレーション。

- 毎日の日次処理で共通ルール＋ステージ個別ルールを適用し、パラメーターを更新
- ステージ進行度が100に達し、遷移条件を満たすと次ステージへ進む
- 全8ステージを経て収穫に至る

> 詳細なパラメーター定義・初期値・共通ルール・ステージ別遷移表・収穫結果計算は `strawberry_progress_design.md` を参照。

#### 成長ゲージの表示

- 各マスの下部に成長ゲージを表示する
- シンプルモデル: `growthPoints / maxGrowthPoints` の割合で満ちる
- アドバンスドモデル: `stageProgress / 100` の割合で満ちる（ステージごとにリセット）
- 収穫可能状態（`harvestable`）では満タン表示でゲージを金色にする

| 項目 | 仕様 | 実装 | 理由 |

### 土画像の選択ロジック

`FarmCell`・`CellDetailPage` の土背景画像は、セルの状態と `AdvancedCropState` のフラグに基づいて以下の通り選択する。

| 条件 | 使用画像 |
|------|---------|
| `status === 'empty'` | `soil-empty.png` |
| アドバンスドモデル：`hasRidge && hasMulch` | `soil-ridged_mulched.png`（畝あり＋マルチあり） |
| アドバンスドモデル：`hasRidge && !hasMulch` | `soil-ridged.png`（畝のみ） |
| アドバンスドモデル：`!hasRidge && hasMulch` | `soil-mulched.png`（マルチのみ） |
| 上記以外（シンプルモデル含む） | `soil-tilled.png` |

この選択ロジックは `src/utils/soilImage.ts` の `getSoilImage()` ユーティリティ関数として実装し、両コンポーネントから共有する。

### ステージ1（栽培準備期）のアクション順序制約

ステージ1内のアクションには現実の農作業に基づく前後関係がある。

| アクション | 実行可能条件 | 不可条件 |
|-----------|-----------|---------|
| 土を整える | `!isTilled` | すでに耕し済み（ボタン非表示） |
| 畝を作る | `!hasRidge` | **`hasMulch` のとき不可**（マルチ後は物理的に困難） |
| マルチを敷く | `!hasMulch` | マルチ済み（ボタン非表示） |
| 元肥を入れる | 常時表示 | **`hasMulch` のとき不可**（土に混ぜ込む作業のためマルチ後は不可） |

**ステージ1の終了条件:** プレイヤーが「苗を植える」アクションを実施する（`isPlanted = true`）

準備アクション（耕す・元肥・畝・マルチ）はすべて**任意**。ただし土を整える（`isTilled`）が苗を植える前提条件となる。

**アクション前後関係（ステージ1）:**

| アクション | 実行可能条件 | 不可条件 |
|-----------|-----------|---------|
| 土を整える | `!isTilled` | `hasMulch`のとき不可（覆われた土は耕せない） |
| 元肥を入れる | `!hasMulch` | `hasMulch`のとき不可 |
| 畝を作る | `!hasRidge && !hasMulch` | `hasMulch`のとき不可 |
| マルチを敷く | **`isTilled && !hasMulch`** | **`!isTilled`のとき不可**（耕す前はマルチを敷けない） |
| **苗を植える** | `isTilled` | `!isTilled`のとき不可 |

**推奨作業順序:** 土を整える → 元肥を入れる → 畝を作る → マルチを敷く → 苗を植える（→ ステージ2へ即時遷移）

| 準備パス | 使用土画像 | 備考 |
|---------|----------|------|
| 耕す → 畝 → マルチ → 苗（推奨） | `soil-ridged_mulched.png` | 最も充実した準備 |
| 耕す → 畝 → 苗 | `soil-ridged.png` | マルチなしで植え付け |
| 耕す → マルチ → 苗 | `soil-mulched.png` | 平畝マルチ栽培 |
| 耕す → 苗（最低限） | `soil-tilled.png` | 準備なしで植え付け（育ちにくい） |

> マルチは耕した後のみ敷設可能。これにより「耕す前にマルチを敷いて詰む」パスを設計上排除する。

**遷移タイミング:** `actionPlantSeedling` で即時ステージ2へ遷移（`pendingStageTransition: true` でモーダル表示）。日次処理でのフォールバックとして `isPlanted` を確認して遷移。

## 13. イベントシステム

### 13.1 概要

イベントには2つの仕組みがある。

**本番イベント（ランダムイベント）:**
日次処理時に確率テーブルに基づいて自動発生する天候・害虫・病気等のイベント。
いちご栽培の各ステージごとのイベント一覧と効果は `strawberry_progress_design.md` のステージ別ランダムイベント表を参照。

**デバッグイベントメニュー:**
畑に様々なイベントを手動でシミュレーション発生させるデバッグ機能。
メイン画面のアクションエリアに「⚡ イベント」ボタンを設置し、専用のイベントメニューページへ遷移する。
メニュー項目は `strawberry_progress_design.md` のランダムイベントに準拠する。

### 13.2 画面構成

### 13.3 イベント一覧

`strawberry_progress_design.md` のステージ別ランダムイベント表に準拠する。
ステージによって同じイベントでも効果値が異なる。デバッグメニューでは代表的な値を適用する。

#### 天候イベント

| イベント名 | アイコン | 効果 | 種別 |

| 晴天続き | ☀️ | 水分 -8 | 中 |
| 日照不足 | 🌫️ | 進行度 -5〜8相当、（成熟期: 色づき低下） | 悪 |
| 強風 | 💨 | ストレス +2 | 悪 |
| 乾燥 | 🏜️ | 水分 -10、（果実肥大期: 実の大きさ低下） | 悪 |

### 13.5 実装フェーズ

- **Phase 1（現在）**: ボタンとメニューページの表示のみ。イベント発動ボタンは「近日実装予定」として表示
- **Phase 2**: ランダムイベントの自動発生を日次処理に組み込む。`strawberry_progress_design.md` のイベント仕様に従う
- **Phase 3**: デバッグメニューからのイベント手動発動を実装（イベント効果を `useGameState` に反映）

### 13.6 天候エフェクト

イベント発動時に FarmPage の畑エリア（ヘッダーとアクションボタンの間）に天候オーバーレイを表示する。

| エフェクト | 対象イベント | 表示方法 | 表示期間 |
|-----------|------------|---------|---------|
| 雨 | rain | Canvas描画による雨粒アニメーション（リアルな降雨表現） | 日付変更まで |
| 長雨 | longRain | Canvas描画による激しい雨粒 + 暗い背景 | 日付変更まで |
| 高温 | highTemp | Canvas描画による陽炎（heat haze）+ 日差し光線 + 画面全体のオレンジ暖色オーバーレイ | 日付変更まで |
| 害虫発生 | pest | Canvas描画による小さな虫が画面内を不規則に飛び回るアニメーション | 日付変更まで |
| 鳥害 | birdDamage | Canvas描画による鳥の飛来アニメーション | 日付変更まで |

- `GameState.activeWeatherEffect` に発動中のイベントIDを保存し、日付変更時にクリアする
- Canvas を使った60fpsの滑らかなアニメーションで描画する
