# バーチャル栽培（Virtual Saibai） — 仕様・要件定義書

## 1. プロダクト概要

### 1.1 コンセプト

AIを活用し、様々な作物（野菜・果実）をバーチャルに栽培できるシミュレーションアプリ。
ユーザーは畑を耕し、種を植え、水や肥料を与えながら作物を育て、収穫を楽しむことができる。

**最大の特徴:** 収穫した作物は実際にユーザーのもとへ届けられる。バーチャルな栽培体験がリアルな報酬につながるユニークなサービスモデル。

### 1.2 プロダクトのゴール

- **フロントエンドプロトタイプ**の構築（栽培画面のみ）
- GitHub Pages で公開し、URLから誰でもアクセスできる形式にする

### 1.3 対象ユーザー

- カジュアルゲームを楽しむ一般ユーザー
- 農業・栽培に興味のあるユーザー

## 2. 技術要件

### 2.1 技術スタック

| 項目 | 技術 |
|------|------|
| フレームワーク | React + TypeScript |
| ビルドツール | Vite |
| スタイリング | Tailwind CSS |
| 状態管理 | React useState / useReducer（ローカルステート） |
| デプロイ | GitHub Pages（`main` ブランチからデプロイ） |
| パッケージ管理 | npm |

> **将来展望:** スマホアプリとしてのリリースを想定。React Native / Capacitor 等での展開を見据え、コンポーネント設計・状態管理ロジックの再利用性を意識する。

### 2.2 動作環境

- モダンブラウザ（Chrome, Firefox, Safari, Edge の最新版）
- レスポンシブ対応（モバイル・タブレット・デスクトップ）

### 2.3 制約事項

- フロントエンドのみ（バックエンド・データベースなし）
- データはブラウザの `localStorage` に保存
- プロトタイプ段階のため、実際の課金処理は実装しない（UIモックのみ）

## 3. 機能要件

### 3.1 栽培画面（メイン画面）

アプリのメイン画面。畑の状態と作物の成長を視覚的に表示し、ユーザーが栽培アクションを実行できる。

#### 3.1.1 画面構成

```
┌─────────────────────────────────────┐
│          ヘッダーエリア              │
│   アプリ名 / ステータス情報           │
├─────────────────────────────────────┤
│                                     │
│          畑・作物表示エリア           │
│   （畑のグリッド + 作物のビジュアル）  │
│                                     │
├─────────────────────────────────────┤
│        アクションボタンエリア         │
│ [耕す] [植える] [水やり] [肥料] [収穫]│
├─────────────────────────────────────┤
│     [🛒 ショップへ]（ページ遷移）     │
└─────────────────────────────────────┘
```

#### 3.1.2 UIコンポーネント一覧

| コンポーネント名 | 説明 |
|-----------------|------|
| `Header` | アプリ名、肥料在庫数、収穫リストへのリンクなどのステータス表示 |
| `FarmGrid` | 畑のマス目をグリッド形式で表示（初期: 2×2 = 4マス） |
| `FarmCell` | 畑の個別マス。状態に応じてビジュアルが変化する。成長ゲージを内包 |
| `GrowthGauge` | 各マスに表示する成長進捗ゲージ |
| `CropDisplay` | マス内の作物の成長段階を表示（AI生成画像） |
| `ActionButtons` | 栽培アクションボタン群 |
| `ShopButton` | ショップ画面へ遷移するボタン |
| `ShopPage` | アイテム購入用の独立ページ |
| `HarvestListPage` | 収穫済み作物の一覧と交換ボタンを表示する静的ページ |
| `NotificationArea` | アクション結果のフィードバックメッセージ表示 |

### 3.2 ユーザーアクション

ユーザーが実行できるアクションの一覧と、各アクションの仕様。

#### 3.2.1 土を耕す

| 項目 | 内容 |
|------|------|
| 対象 | 未使用状態のマス |
| 前提条件 | マスが「未使用」状態であること |
| 結果 | マスの状態が「耕された土」に変化 |
| 表示変化 | 茶色い耕された土のビジュアルに変化 |

#### 3.2.2 種を植える

| 項目 | 内容 |
|------|------|
| 対象 | 耕された状態のマス |
| 前提条件 | マスが「耕された」状態であること |
| 操作 | 作物を選択して植える（作物選択UIを表示） |
| 結果 | マスの状態が「種まき済み」に変化 |
| 表示変化 | 種のアイコンが表示される |

**選択可能な作物（プロトタイプ）:**

プロトタイプでは1〜2種類に絞る。

| 作物 | 成長段階数 | 収穫までのアクション回数目安 |
|------|-----------|------------------------|
| トマト | 4 | 水やり8回 + 肥料2回（目安） |
| いちご | 4 | 水やり8回 + 肥料2回（目安） |

> 成長ゲージ追加に伴い、1回あたりの成長量を調整し適度な育てごたえを確保する。詳細は 5.3 参照。

#### 3.2.3 水をやる

| 項目 | 内容 |
|------|------|
| 対象 | 種まき済み or 成長中のマス |
| 前提条件 | マスに作物が植えられていること |
| 結果 | 作物の成長ポイントが加算される |
| 表示変化 | 水滴アニメーション + 成長段階が進む場合はビジュアル変化 |

#### 3.2.4 肥料をまく

| 項目 | 内容 |
|------|------|
| 対象 | 種まき済み or 成長中のマス |
| 前提条件 | マスに作物が植えられていること、肥料の在庫があること |
| 結果 | 作物の成長ポイントが大幅に加算される（水やりの2倍） |
| 表示変化 | 肥料散布アニメーション + 成長段階が進む場合はビジュアル変化 |
| 消費 | 肥料アイテムを1つ消費 |

#### 3.2.5 収穫する

| 項目 | 内容 |
|------|------|
| 対象 | 作物が完全に実ったマス |
| 前提条件 | 作物が最終成長段階に到達していること |
| 結果 | 作物を収穫。マスは「耕された」状態に戻る。収穫した作物は「収穫リスト」に記録される |
| 表示変化 | 収穫アニメーション + 収穫おめでとう演出 |
| 報酬 | 収穫した作物が実際にユーザーへ届けられる（本番想定） |

> **プロトタイプでの扱い:** 実際の配送処理は実装しない。収穫時に「おめでとう！トマトが届きます」のような演出を表示し、収穫履歴として記録するのみ。将来的にはEC/物流システムとの連携を想定。

### 3.3 収穫リスト機能

ヘッダーの収穫アイコンから遷移する独立ページ（`HarvestListPage`）。

#### 3.3.1 画面構成

```

│  [← 畑に戻る]   収穫リスト    │
├──────────────────────────────┤
│  収穫した作物の一覧            │
│                              │
│  🍅 トマト  2026/03/16  [交換]│
│  🍓 いちご  2026/03/15  [交換]│
│  🍅 トマト  2026/03/14  [交換]│
│                              │
│  （収穫なしの場合はその旨表示） │
```

#### 3.3.2 交換ボタン

- 各収穫済み作物の行に「交換」ボタンを配置する
- プロトタイプのため、ボタン押下時は「交換申請を受け付けました」等のメッセージを表示するのみ（実際の配送処理なし）

## 4. データモデル

プロトタイプ段階では `localStorage` にJSON形式で保存する。

### 4.1 ゲーム状態（GameState）

```typescript
interface GameState {
  fertilizer: number;         // 肥料の在庫数
  farmSize: number;           // 畑のマス数
  cells: FarmCellState[];     // 各マスの状態
  harvestLog: HarvestRecord[]; // 収穫履歴
}

interface HarvestRecord {
  crop: CropType;
  harvestedAt: string;        // ISO 8601 日時
}
```

### 4.2 畑マスの状態（FarmCellState）

```typescript
interface FarmCellState {
  id: number;
  status: CellStatus;
  crop: CropType | null;
  growthPoints: number;     // 現在の成長ポイント
  maxGrowthPoints: number;  // 最大成長ポイント（収穫可能になる値）
  growthStage: number;      // 現在の成長段階（表示用）
  maxGrowthStage: number;   // 最大成長段階
}
```

### 4.3 列挙型

```typescript
type CellStatus = 'empty' | 'tilled' | 'planted' | 'growing' | 'harvestable';

type CropType = 'tomato' | 'strawberry';
```

### 4.4 作物定義（CropDefinition）

```typescript
interface CropDefinition {
  type: CropType;
  name: string;              // 表示名
  maxGrowthPoints: number;   // 収穫に必要な成長ポイント
  growthStages: number;      // 成長段階数
  stageImages: string[];     // 各成長段階のAI生成画像パス (例: ['/assets/tomato-1.png', ..., '/assets/tomato-4.png'])
}
```

## 5. 画面遷移・状態遷移

### 5.1 画面遷移

```
栽培画面（メイン） ⇄ ショップページ
栽培画面（メイン） ⇄ 収穫リストページ
```

3画面構成。React Router（HashRouter）を使ったページ遷移。

### 5.2 畑マスの状態遷移

```
[empty] ──(耕す)──▶ [tilled] ──(種を植える)──▶ [planted]
                       ▲                          │
                       │                    (水やり/肥料)
                       │                          ▼
                  (収穫する)◀── [harvestable] ◀── [growing]
                                              (成長ポイント
                                               が最大に到達)
```

### 5.3 成長ロジック

#### 成長ポイントの加算

| アクション | 加算量 |
|-----------|--------|
| 水やり | +1 pt |
| 肥料 | +3 pt |

#### 作物の成長パラメータ

| 作物 | 最大成長ポイント（収穫に必要） | 成長段階数 | 各段階の区切りポイント |
|------|--------------------------|-----------|---------------------|
| トマト | 12 pt | 4 | 3 / 6 / 9 / 12 |
| いちご | 12 pt | 4 | 3 / 6 / 9 / 12 |

> 水やり12回、または水やり9回+肥料1回などで収穫可能になる。肥料を活用すると効率的に育てられる。

#### ステージ進行ルール

1. 成長ポイントが各段階の区切りポイントに達するたびに `growthStage` が1つ進む
2. `growthStage` が `maxGrowthStage`（4）に達すると `harvestable` 状態になる
3. 成長ポイントは上限（`maxGrowthPoints`）を超えない

#### 成長ゲージの表示

- 各マスの下部に成長ゲージを表示する
- ゲージは `growthPoints / maxGrowthPoints` の割合で満ちる
- 次のステージまでの残りポイントが視覚的に分かるよう、各段階の区切りにマーカーを表示する
- 収穫可能状態（`harvestable`）では満タン表示でゲージを金色にする

## 6. UI/UXデザイン方針

### 6.1 ビジュアルスタイル

- **テイスト**: 温かみのある、カジュアルなイラスト風
- **配色**: 緑・茶・青を基調とした自然をイメージしたカラーパレット
- **作物の表現**: 画像生成AIで作成したイラストを使用。各作物の成長段階ごとに画像を用意する

### 6.2 作物画像アセット仕様

画像生成AIを使用して、各作物の成長段階ごとのイラストを作成する。

| 項目 | 仕様 |
|------|------|
| フォーマット | PNG（透過背景推奨） |
| サイズ | 256×256px（表示時はCSSでスケール） |
| テイスト | 統一されたイラスト調（リアル寄りよりもカジュアルなイラスト風） |
| 命名規則 | `{作物名}-{段階番号}.png`（例: `tomato-1.png`） |

**必要な画像一覧:**

| 作物 | 段階1（種/芽） | 段階2（苗） | 段階3（成長） | 段階4（実り） |
|------|--------------|------------|-------------|-------------|
| トマト | 種まき直後の芽 | 小さな苗 | 花が咲いた状態 | 赤いトマトが実った状態 |
| いちご | 種まき直後の芽 | 葉が広がった苗 | 白い花が咲いた状態 | 赤いいちごが実った状態 |

### 6.3 カラーパレット

| 用途 | カラーコード | 説明 |
|------|------------|------|
| 背景 | `#F5F0E8` | 温かみのあるベージュ |
| 畑の土 | `#8B6914` | 耕された土の茶色 |
| 未使用の土 | `#A0926B` | グレーがかった茶色 |
| アクセント | `#4CAF50` | 植物の緑 |
| ボタン（プライマリ） | `#2E7D32` | 濃い緑 |
| ボタン（課金） | `#FF9800` | オレンジ |
| テキスト | `#333333` | ダークグレー |
| コイン | `#FFD700` | ゴールド |

### 6.4 レスポンシブ対応

| 画面サイズ | レイアウト |
|-----------|-----------|
| モバイル（〜480px） | 畑グリッド: 2列、アクションボタン: 縦並び |
| タブレット（481px〜768px） | 畑グリッド: 3列、アクションボタン: 横並び |
| デスクトップ（769px〜） | 畑グリッド: 4列、アクションボタン: 横並び |

### 6.5 フィードバック

- アクション実行時にトースト通知で結果を表示（例: 「水をやりました！🌱」）
- 無効なアクション時はボタンを非活性にし、理由をツールチップで表示
- 成長段階が進む際に軽いアニメーション効果

### 6.6 レイヤー（z-index）管理

| レイヤー | 要素 | 対応方針 |
|---------|------|----------|
| 最前面 | モーダル（CropSelector, リセット確認）、トースト通知 | z-50 以上 |
| 前面 | アクションボタンエリア（sticky bottom） | z-30 |
| 中間 | マスの「選択中」インジケータ | z-20 |
| 背面 | マスのステータスラベル（「空き地」等） | z-10、アクションボタンに隠れる場合は非表示 |

> アクションボタンエリアがスクロールで畑マスと重なった場合、マスのステータスラベルはアクションボタンの下に隠れる（クリッピングしない）。ラベルがボタンに被さって前面に出ないよう z-index を管理する。

### 6.7 ヘッダーアイコンの表記

ヘッダーのステータス表示は、アイコンだけでは意味が分かりにくいため、ラベルを明示する。

| 表示内容 | 現状 | 改善後 |
|---------|------|--------|
| 肥料在庫 | 🧪 3 | 🧪 肥料 3個 |
| 収穫済み作物 | 📦 2 | 📦 収穫済み 2個（タップで収穫リストへ遷移） |

## 8. デプロイ要件

- `main` ブランチから GitHub Actions を使って自動ビルド&デプロイ
- `vite.config.ts` に `base` パスを設定（リポジトリ名に対応）
