# いちご栽培のルール

いちごを選択し栽培する際の、成長ルール（管理するパラメーター、成長段階、ユーザーアクション、イベント）を定義する。

## 1. 前提

### 時間単位

* 1日 = 1ターン

### アクション回数

* 1ターンのアクション回数は **無制限**
* 現実的に可能かどうかを基準とする（水やりを何度もやるのは可能だが、土壌が水分100に達したらそれ以上は選択不可）
* 各パラメーターの上限（0〜100）が自然な行動制限として機能する

### 日送り

* 1日の区切りは **現実の時間（日付変更）** に準拠する
* プレイヤーは毎日、好きなタイミングでアクションを実行する
* 現実の日付が変わると日次処理（状態遷移・更新）が実行される
* 未ログイン日がある場合は、未ログイン日数分の日次処理を **すべて反映** する（天候イベントも日数分ロールする）
* 栽培上の時間変化（○月、季節など）は現実時間とは別にゲーム内で管理する

### アクションの程度選択

水やり・追肥など量が関わるアクションは **程度を選択** できる。
「適量」「多すぎ」を別アクションとして並べるのではなく、プレイヤーが程度を選び、結果としてパラメーターが変動する。

例: 水やりの場合
* 少なめ: 水分 +8
* ふつう: 水分 +15
* たっぷり: 水分 +25

水分が100に達した場合はそれ以上の水やりは選択不可。
日末の水分値で評価し、適正範囲（40〜69）かどうかが健全度やステージ進行度に影響する。

### 即時反映と日次反映

* 耕す・畝を作る・ビニールシートを敷くなど物理的な作業は **即時に状態へ反映** される
* 植物の成長・パラメーターの増減による評価は **日次処理で反映** される

### 1日の処理順

1. 天候・ランダムイベント決定
2. プレイヤー行動を反映
3. 状態値を更新
4. 病気・害虫・乾燥・過湿などを判定
5. ステージ進行度を更新
6. ステージ遷移判定
7. 収穫可能なら収穫可表示

この順番にしておくと、「その日に何をしたか」が分かりやすく結果に反映されます。

## 3. 共通ルール

### 毎日の自然変化

これらは全ステージ共通のベースです。**上書き方式** で適用します。

* 水分: -5
* 雑草量: +2
* 害虫リスク: +1
* 病気リスク: +1

#### 上書き方式のルール

* 共通ルールは全ステージで常にベースとして適用される
* ステージ個別に同じ項目が記載されている場合は、その値で **上書き** する（加算ではない）
* ステージ個別に共通にない項目が記載されている場合は、その値を **追加** する
* ステージ個別に記載がない項目は、共通ルールの値がそのまま適用される

例: ステージ6の場合
* 水分: 共通 -5 → ステージ6で -7 と記載 → **-7を適用**（上書き）
* 雑草量: ステージ6に記載なし → **+2を適用**（共通が生きる）
* 害虫リスク: ステージ6に記載なし → **+1を適用**（共通が生きる）
* 病気リスク: ステージ6に記載なし → **+1を適用**（共通が生きる）
* 腐りやすさ: ステージ6独自で +1 → **+1を追加**（共通にはない項目）

### 健全度の更新

1日の最後に以下を反映します。

* 水分が 40〜69 なら 健全度 +1
* 栄養が 40〜69 なら 健全度 +1
* ストレスが 60以上なら 健全度 -3
* 病気リスクが 60以上なら 健全度 -4
* 害虫リスクが 60以上なら 健全度 -3
* 腐りやすさが 60以上なら 健全度 -2

### ストレスの増減

* 高温イベント: +8
* 低温イベント: +8
* 水分 24以下: +6
* 水分 85以上: +5
* 温度調整成功: -5

### 枯死条件

* 健全度が 0 以下で枯れる

## ステージ5: 開花期

### 目的

花を実に変える。

### 新しい考え方

この段階では「受粉成功率」を内部で使います。
これは“花がどれだけ実になるかの割合”です。

#### 基本の受粉成功率

* 基本 50%

#### 補正

* 受粉補助あり: +20%
* 長雨: -20%
* 高温: -15%
* 病気リスク 60以上: -10%

最終的に、その日に実になる数は
`その日対象の花の数 × 受粉成功率`
で決めます。

### 主な行動と効果

| 行動       | 変化               | 理由              | 表示           |
| -------- | ---------------- | --------------- | ------------ |
| 水やり（適量）  | 水分 +12、健全度 +2    | 開花中の株を支えるため     | 花が元気         |
| 水やり（多すぎ） | 水分 +25、病気リスク +5  | 湿気で病気が出やすいため    | 花まわりが湿りすぎ    |
| 軽い追肥     | 栄養 +5            | 開花中の負担を支えるため    | 少し肥料を足した     |
| 受粉補助     | 受粉成功率 +20%       | 実がつきやすくなるため     | 花が実になりやすくなった |
| 花を減らす    | 花の数 -2〜5、品質補正 +5 | 数を絞ると残りに資源が回るため | 花数を調整した      |

## 5. 収穫結果の決定

ここはユーザーから見て最も重要なので、分かりやすくします。

### 表示する結果

* 収穫個数
* 総重量
* 平均果重
* 品質ランク
* 甘さ評価
* 見た目評価
* 病気・虫害の影響
* コメント

---

### 収穫個数

基本はそのまま `実の数`

例:

* 実の数 = 12
  → 収穫個数 = 12個

---

### 総重量

`総重量 = 実の数 × 基本重さ × 大きさ補正 × 健全度補正`

#### 推奨値

* 基本重さ = 15g
* 大きさ補正 = 0.5 + 実の大きさ / 100
* 健全度補正 = 0.5 + 健全度 / 200

#### 例

* 実の数 = 12
* 実の大きさ = 80
* 健全度 = 70

なら

* 大きさ補正 = 1.3
* 健全度補正 = 0.85

総重量 = 12 × 15 × 1.3 × 0.85 = 約199g

---

### 平均果重

`平均果重 = 総重量 ÷ 収穫個数`

上の例なら

* 約16.6g/個

---

### 品質スコア

`品質スコア = 50 + 甘さ補正 + 色づき補正 + 健全度補正 - 病害虫減点 - 腐敗減点 + 管理加点`

#### 補正式の例

* 甘さ補正 = 甘さ × 0.3
* 色づき補正 = 色づき × 0.2
* 健全度補正 = 健全度 × 0.1
* 病害虫減点 = 品質被害蓄積
* 腐敗減点 = 腐りやすさ × 0.2
* 適期収穫なら +10
* 花や実の調整を適切に行っていたら +5

#### ランク

* 85以上: A
* 70以上: B
* 50以上: C
* 49以下: D

---

### 甘さ評価表示

* 0〜29: あまり甘くない
* 30〜49: ほどほど
* 50〜69: 甘い
* 70〜100: とても甘い

### 見た目評価表示

見た目は以下でまとめると分かりやすいです。

* 色づき
* 病害虫被害
* 腐りやすさ

例:

* 見た目: とても良い
* 見た目: 少し傷あり
* 見た目: 傷みが目立つ

### 評価コメント

品質ランクに応じた感想テキストを表示し、プレイヤーの達成感と次回への意欲につなげる。

#### ランク別コメント例

| ランク | コメント例                                  |
| --- | -------------------------------------- |
| A   | 「素晴らしい出来です！プロ農家も驚くほどの仕上がりですね」         |
| B   | 「よく育てましたね！あと少しの工夫でさらに良くなりそうです」        |
| C   | 「無事に収穫できました。いくつか改善ポイントがあります」          |
| D   | 「栽培は難しいですよね。次回はポイントを押さえてリベンジしましょう」    |

#### 改善ヒント

品質スコアの各補正値を分析し、最もマイナスに効いた要因から **最大2つ** のヒントを表示する。

| 判定条件              | ヒント例                                    |
| ----------------- | --------------------------------------- |
| 甘さが50未満           | 「成熟期に水を控えめにすると、糖度が上がりやすくなります」           |
| 色づきが70未満          | 「葉の整理で日光を当てると、もっときれいに赤くなります」            |
| 品質被害蓄積が20以上       | 「害虫・病気対策をこまめに行うと、見た目がぐっとよくなります」        |
| 腐りやすさが40以上        | 「水やりを控えめにし、風通しを確保すると傷みにくくなります」         |

| 健全度が50未満          | 「水・栄養・ストレス管理を丁寧にすると、株が元気に育ちます」        |

### 6.3 デイリーアドバイス

日付変更後（ログイン時）に、現在の状態に応じた **1行アドバイス** を画面上部に表示する。

#### アドバイスの優先度（高い順）

2. 現在ステージで重要なアクション（受粉補助、摘果など）
3. パラメーター改善の提案（水やり、除草など）
4. ポジティブなフィードバック（状態が良好な時）

#### 例

* 「⚠️ 株がとても弱っています。水やりと栄養補給を急ぎましょう」

* 「✂️ 実が12個あります。摘果すると1粒が大きく甘くなります」
* 「🍓 実が色づいてきました。水は控えめにすると甘くなります」
* 「✨ 順調に育っています。この調子で管理を続けましょう」

### 6.4 ステージ遷移時の演出

ステージが進んだときに、お祝いメッセージと次ステージの丁寧な説明を表示する。

#### 表示内容

* お祝いアイコン + 「○○期に入りました！」
* 次のステージで何が起こるか
* プレイヤーが気をつけるべきポイント
* 使えるようになるアクション（あれば）

#### 各ステージ遷移時のメッセージ例

**ステージ1→2（定植・活着期）**
> 🎉 畑の準備ができました！いよいよ苗を植えましょう。
> この時期は苗がしっかり根付くことが大切です。
> 水を切らさないよう注意しつつ、やりすぎにも気をつけましょう。
> 寒さや暑さのストレスにも敏感な時期です。

**ステージ2→3（葉成長期）**
> 🎉 苗が根付きました！ここからぐんぐん育ちます。
> 葉が茂って株が大きくなる時期です。
> 水やり・追肥・除草をバランスよく行いましょう。
> 葉の整理で風通しをよくすると病気予防になります。

**ステージ3→4（花芽形成期）**
> 🎉 株がしっかり育ちました！花の準備が始まります。
> この時期の水・栄養・温度のバランスが、花の数を決めます。
> 温度ストレスは花芽形成に大きく影響するので、温度調整が重要です。

**ステージ4→5（開花期）**
> 🌸 花芽ができました！白い花が咲き始めます。
> 花が実になるかどうかは受粉にかかっています。
> 「受粉補助」をすると実がつきやすくなります。
> 雨が多いと受粉しにくくなるので注意しましょう。

**ステージ5→6（果実肥大期）**
> 🎉 実がつき始めました！ここから実を大きく育てます。
> 水と栄養をしっかり与えて、実をふくらませましょう。
> 実が多すぎると1粒が小さくなります。摘果も検討しましょう。

**ステージ6→7（成熟期）**
> 🎉 実が大きく育ちました！あとは甘く色づくのを待ちます。
> この時期は水を控えめにすると糖度が上がります。
> 葉の整理で日光を当てると色づきがよくなります。
> やりすぎない、見守る管理が大切な時期です。

**ステージ7→8（収穫可能期）**
> 🍓 いちごが赤く実りました！いよいよ収穫です！
> 収穫のタイミングが大切です。待ちすぎると熟しすぎて傷みます。
> 天候にも注意し、良い状態のうちに収穫しましょう。

### 6.5 見た目変化の例

* 水分不足 → 葉がしおれる
* 過湿 → 葉色が悪い、土がぬれている
* 病気発生 → 葉や実に斑点
* 害虫発生 → 葉に穴、虫アイコン
* 雑草が多い → 足元に雑草が見える
* 開花期 → 白い花が増える
* 果実肥大期 → 実が大きくなる
* 成熟期 → 赤くなる
* 収穫遅れ → 熟れすぎた見た目

## 7. ルールの読み方

この完全版では、各効果を以下の考え方で統一しています。

### 行動

プレイヤーが意図して環境を調整するもの。
直接の良い効果があるが、やりすぎると副作用もある。

例:

* 水やり → 水分を増やす
* ただし多すぎる → 病気や腐敗につながる

### ランダムイベント

自然環境の変化。
複数の状態値を同時に変える。

例:

* 長雨 → 水分増加、病気増加、熟期なら腐敗増加

### 品質

1回の行動では決まらず、日々の管理結果の積み重ねで決まる。
